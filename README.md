# Sky Ratlandia
A collaborative Minecraft modpack managed with [packwiz](https://packwiz.infra.link/).

- **Minecraft:** 1.21.1
- **Loader:** NeoForge 21.1.95
- **Pack format:** packwiz

## What's in this repo

```
.
├── pack.toml            # Pack metadata (MC + loader versions)
├── index.toml           # Hash index of every file in the pack (auto-generated)
├── mods/                # One .pw.toml per mod (auto-generated by packwiz)
├── config/              # Mod config files — commit these, this is your customization
├── resourcepacks/       # Optional resource packs
├── shaderpacks/         # Optional shader packs
├── scripts/build.py     # Builds client (.mrpack) + server (.zip) artifacts
└── .github/workflows/   # CI that runs scripts/build.py on every push
```

Mod jars are **not** committed. Each `mods/*.pw.toml` is a tiny manifest pointing at a Modrinth/CurseForge ID + hash, so the repo stays small.

## First-time setup

1. Install packwiz: <https://packwiz.infra.link/installation/>
2. Clone this repo:
   ```sh
   git clone <repo-url>
   cd <repo-name>
   ```
3. Install a launcher that supports packwiz import:
   - **Prism Launcher** (recommended) — paste the URL of a hosted `pack.toml` to import
   - **MultiMC** — same as Prism
   - **Modrinth App** — import the `.mrpack` produced by CI
   - **CurseForge App** — import the CurseForge zip produced by CI

## Adding a mod

```sh
# From Modrinth (preferred — no API key needed)
packwiz mr install sodium

# From CurseForge
packwiz cf install jei

# Refresh hashes after changes
packwiz refresh
```

Then commit the new `mods/<slug>.pw.toml` and the updated `index.toml`.

## Removing a mod

```sh
packwiz remove <slug>
packwiz refresh
```

## Editing a config

Drop or edit a file under `config/`. Then:

```sh
packwiz refresh
```

Commit the changed config and the updated `index.toml`.

## Workflow

1. `git checkout -b <type>/<short-description>` — see CONTRIBUTING.md for naming
2. Make your changes, run `packwiz refresh`
3. `git commit -am "..."` and push
4. Open a PR — describe what changed and (if a mod) why
5. A maintainer reviews and merges
6. CI builds a fresh `.mrpack` + CurseForge zip and attaches them to the run

## Building locally

The pack ships a self-contained Python build script — no packwiz install needed:

```sh
python scripts/build.py
```

Output lands in `build/<name>-<version>.mrpack` — drop it into the
**Modrinth App** or **Prism Launcher** to install. Downloaded jars are
cached in `.build_cache/` so re-runs are fast; pass `--clean` to wipe
`build/` first.

CI runs the same script on every push and uploads the artifact. Tagging a
commit with `v*` (e.g. `v0.2.0`) also publishes it as a GitHub release.

### packwiz-style export (optional)

If you want the canonical packwiz exports for distribution to packwiz-aware
launchers, install packwiz (`go install github.com/packwiz/packwiz@latest`) and:

```sh
packwiz modrinth export      # produces a .mrpack via packwiz's own logic
packwiz curseforge export    # CurseForge zip
```

## Versioning

Bump `version` in `pack.toml` whenever you publish a release. We use SemVer-ish:
`MAJOR.MINOR.PATCH` — major for MC/loader version bumps, minor for new mods,
patch for config tweaks and version bumps of existing mods.
